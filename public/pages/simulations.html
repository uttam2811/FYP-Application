About This Simulation

The GA Optimised Weather Prediction simulation combines a Genetic Algorithm (GA)-based optimization engine with high-resolution weather forecasting to determine optimal dispatch scheduling for the IEEE 13 Bus microgrid with distributed solar PV generation.

The system operates in a two-stage pipeline:

  • Stage 1 — Weather & Irradiance Generation: A realistic 24-hour weather profile is generated at 15-minute intervals (96 time steps). The model includes smooth solar production curves (sine-based), stochastic cloud variation, Gaussian noise injection, abnormal spike correction, and moving-average smoothing to produce physically plausible irradiance and temperature profiles.
  • Stage 2 — GA Optimisation: The genetic algorithm takes the forecasted irradiance and temperature profiles as inputs and optimizes the power dispatch across all distributed generators (DGs), energy storage systems, and grid connection points. The GA uses tournament selection, crossover, and mutation operators to evolve a population of candidate solutions toward the Pareto-optimal front.

The Simulink model implements the full IEEE 13 bus test feeder topology including voltage regulators (RG60), substation transformer (XFXFM1), distributed loads (Y-connected, Delta-connected, and constant PQ), three-phase line segments with various configurations (601–607), and solar PV subsystems with MPPT controllers.

Key outputs include bus voltage profiles, line power flows, total system losses, PV generation curves, GA convergence plots, and optimal scheduling tables.

Common Weather + PV Irradiance Generator

About This Code

This MATLAB script is the common weather preprocessing module shared by all three simulations. It generates a realistic 24-hour weather forecast with 15-minute resolution (96 data points) that serves as input to the Simulink models.

The pipeline includes:

  • Irradiance Model: Sine-based solar curve (6 AM – 6 PM) with stochastic cloud variation
  • Temperature Model: Day-night sinusoidal variation with Gaussian noise
  • Spike Correction: Detects and interpolates abnormal jumps in both irradiance (>300 W/m²) and temperature (>5°C)
  • Smoothing: 3-point moving average filter for physically plausible profiles
  • Output: Saves weather_data_advanced.mat with time-series arrays formatted for Simulink lookup tables
MATLAB weather_optimization.m 
%% ===================================================
% ADVANCED WEATHER + PV IRRADIANCE GENERATOR
% IEEE 13 BUS MICROGRID
% 15-min resolution + Spike Correction + Smoothing
%% ===================================================

clear; clc; close all;

disp('Generating high-resolution 24-hour weather forecast...');

%% ===================================================
%% 1. TIME SETTINGS (15-minute resolution)
%% ===================================================

dt = 900;                     % 15 minutes in seconds
n_hours = 24;

time_seconds = (0:dt:(n_hours*3600 - dt))';
n_points = length(time_seconds);
time_hours = time_seconds / 3600;

%% ===================================================
%% 2. IRRADIANCE MODEL (Realistic + Cloud Variation)
%% ===================================================

irradiance = zeros(n_points,1);

for i = 1:n_points
    
    hour = time_hours(i);
    
    if hour >= 6 && hour <= 18
        
        % Smooth solar production curve
        base_curve = 1000 * sin(pi*(hour-6)/12);
        
        % Slowly varying cloud effect
        cloud_variation = 0.85 ...
                        + 0.15*sin(0.4*hour) ...
                        + 0.05*randn;
        
        irradiance(i) = base_curve * cloud_variation;
        
    else
        irradiance(i) = 0;
    end
end

% Ensure no negative irradiance
irradiance = max(irradiance,0);

%% ===================================================
%% 3. TEMPERATURE MODEL (Day-Night Variation + Noise)
%% ===================================================

temperature = 28 + 6*sin(pi*(time_hours-6)/12);
temperature = temperature + 0.5*randn(size(temperature));

% Minimum realistic temperature limit
temperature = max(temperature,20);

%% ===================================================
%% 4. SPIKE CORRECTION (Abnormal Jump Removal)
%% ===================================================

% ---- IRRADIANCE SPIKE FIX ----
for i = 2:n_points-1
    if abs(irradiance(i) - irradiance(i-1)) > 300
        irradiance(i) = (irradiance(i-1) + irradiance(i+1))/2;
    end
end

irradiance = max(irradiance,0);

% ---- TEMPERATURE SPIKE FIX ----
for i = 2:n_points-1
    if abs(temperature(i) - temperature(i-1)) > 5
        temperature(i) = (temperature(i-1) + temperature(i+1))/2;
    end
end

temperature = max(temperature,20);

%% ===================================================
%% 5. FINAL SMOOTHING (Moving Average Filter)
%% ===================================================

irradiance = movmean(irradiance,3);
temperature = movmean(temperature,3);

%% ===================================================
%% 6. FORMAT FOR SIMULINK
%% ===================================================

irradiance_profile = [time_seconds irradiance];
temperature_profile = [time_seconds temperature];

%% ===================================================
%% 7. SAVE FILE
%% ===================================================

save('weather_data_advanced.mat',...
     'irradiance_profile',...
     'temperature_profile');

disp('Advanced weather file created successfully.');

%% ===================================================
%% 8. OPTIONAL PLOT (For Verification)
%% ===================================================

figure;

subplot(2,1,1)
plot(time_hours,irradiance,'LineWidth',1.5)
xlabel('Time (Hours)')
ylabel('Irradiance (W/m^2)')
title('24-Hour Irradiance Profile')
grid on

subplot(2,1,2)
plot(time_hours,temperature,'LineWidth',1.5)
xlabel('Time (Hours)')
ylabel('Temperature (°C)')
title('24-Hour Temperature Profile')
grid on

About This Simulation

The 1-Day Power Loss simulation runs the IEEE 13 Node Test Feeder with 3 distributed solar PV subsystems for a full 24-hour period. It analyzes power losses across the distribution network, tracking how power balance changes as solar generation varies throughout the day.

  • Slack Bus (650): Measures total power injected from the substation
  • Load Buses: Measures active and reactive power consumption at each load point
  • PV Buses (634, 671, 675): Tracks solar generation at each PV-connected bus
  • System Losses: Calculated as P_slack + P_PV - P_load across all time steps

Results are streamed live to this dashboard. After completion, voltage profiles, power flows, bus status, and load connection status are displayed in the Live Results and Bus Status tabs.
