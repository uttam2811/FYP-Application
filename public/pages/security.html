What Protects Your Account

Two-step verification (OTP): Login and registration require one-time code validation to reduce account takeover risk.

Secure session cookies: Authentication tokens are stored in HTTP-only cookies so scripts cannot read them directly.

Request throttling: Login and API calls are rate-limited to mitigate brute-force and abuse attempts.

Browser security headers: The app enforces strict security headers to reduce clickjacking and script injection risks.

Recommended User Practices

1. Use a unique password and avoid reusing passwords from other websites.

2. Do not share OTP codes with anyone, including support staff.

3. Log out after finishing work, especially on shared devices.

4. Report suspicious login behavior to the system administrator.
